用 Hugo 架設自己的個人網站 (1)

這系列文章主要放置建立個人網站的筆記，使用到的內容應該會有:

  • Golang (可以先到GO的官方網站下載好最新版本)
  • Hugo

有那麼多記錄的網站，為什麼還要自己架？

為什麼想架個人站呢，因為:

  1. 試過那麼多架個人網站的管道之後，總覺得好像還少了什麼
  2. 覺得不夠漂亮不喜歡(喂

下面是我曾經用過的管道，除了 Notion 的電腦版我拿來當個人生涯筆記之外，現在都比較少用:

  • Medium: 付費有點麻煩，雖然我經常在上面看厲害的大大們發文章。
  • WordPress: 原本想要認真用，但入門感覺要花一點時間，有些功能好像也用不到。
  • HackMD: 一開始是因為模板本身就含有旁邊的文章大綱，可以協助閱讀，寫了實作筆記之後，覺得不如notion使用起來的方便，考慮之後只用來放較短篇幅的東西，或是都放實作相關的紀錄。
  • Notion: 初入門的筆記都寫在這上面，性質比較接近個人的生活筆記本，但我還需要一個可以認真討論、紀錄技術的地方，覺得大雜燴還是跟整理好的內容應該分開。

之前看別人使用 Hexo 架設個人網站，也興起了自己來建一個的念頭，畢竟工程師ㄉ特性之一不就是要創造東西嗎，於是資料找著找著就看見了幾篇比較Jekyll、Hexo、Hugo 的文章，然後相中了 Hugo 超快速的文章生成速度(畢竟是用 Go 寫的)，就決定使用它來架站了!

想寫 Blog 的原因

剛接觸前端這個領域的時候，就維持著學習搭配做筆記的習慣，對我來說，寫筆記、文章的最大好處，是可以在遇到相同問題時，減少又要重新蒐集、整理資料的時間，例如使用框架，別人可能還在依照官方網站或不停餵狗來獲取資訊，你就可以翻出筆記、優雅地依照需求，尋找你需要的內容來使用。

看了文章，其實跟目前作筆記的原因是差不多的，而且我也希望自己能幫到未來的自己(目標是時間管理大師(不對

廢話說ㄌ這麼多，開始進行建置ㄅ!

初始建置

整體架設參考了鐵人賽的系列文章和 Hugo 官方文件，我們可以在任何可以使用 go 語言的機器上安裝 Hugo CLI。

Hugo 環境建置

我是用 chocolatey 安裝 hugo，如果沒有安裝過 chocolatey ，可以先跟著 chocolatey 官方的安裝文件完成安裝，而關於 chocolatey 的環境配置，這裡有一篇文章可以參考。

  • 以系統管理員身分開啟 PowerShell
    1. 檢查系統的指令檔執行限制
    2. 設定執行限制為 Allsigned 或 ByPass
    3. 將安裝指令輸入
    4. 顯示 chocolatey安裝完成
    5. 輸入 choco 檢查版本
  • 安裝 hugo
    1. 輸入 choco install hugo-extended -confirm (未來可能會用到Sass/SCSS，所以安裝 extended 的版本)
    2. 安裝完成
    3. 輸入 hugo version 檢查版本

安裝主題：Stack

跟著官方文件的 quick start 跑就好，我自己是先在本機創了一個個人資料夾，在資料夾裡面新建內容，輸入hugo new site quickstart，接下來的動作都會在 cd 到 quickstart ，這個新增的資料夾內進行。

Stack 這個主題跟官方提供的安裝方法有點不一樣，所以也參考了主題製作者的文件、別人的筆記

#到資料夾裡
cd quickstart
#版本控制
git init
#載入模組
git submodule add https://github.com/CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack

好了之後記得把原本的一個 config.toml 刪除，接著到 themes > hugo-theme-stack > exampleSite，把裡面的檔案複製到 quickstart 裡，接著輸入 hugo server -D 就可以在 localhost:1313 看到預設模板的樣子了。

新增文章

在 quickstart 資料夾裡，輸入 hugo new post/my-first-post.md ，更新 host 之後就會看到新的文章出現，到目前為止，就是最基礎的建置，我們得到了一個套用好範例模板、新增一篇文章的部落格。
