用 Hugo 架設個人網站(2)

關於 hugo 的 config 設置、客製化主題和部署

用 Hugo 架設自己的個人網站 (2)

在前面的文章，已經成功讓套用好模板、發布第一篇新文章的部落格在本地端運行了，接下來可以開始做更多的個人化配置。

啊因為我對版本控制還沒有非常熟悉，有些步驟對老手來說，可能會有較多餘的記錄，可以視情況跳過！

結構 Configuration

首先可以來看一下部落格資料夾的結構：

.
├── archetypes 可以放置文章模板
├── config.yaml 部落格的架構設定
├── content 文章內容會在這裡
├── data 可以放置資料文件來使用
├── layouts 放置各式 view
├── public 靜態網站的資料放置位置
├── resources 生成的檔案檔
├── static 可以放 images、favicon 等靜態檔案
└── themes 使用的主題會放在這裡

跟著主題的文件設定一輪之後，基礎的設定都能改好，像是 url 的呈現格式、dark/light theme 的設定等，基本上很詳盡，也有簡體中文版可以參閱。

客製化主題 Modify theme

我自己是沒有更動 SCSS 裡的 variable.scss，但 Stack 這個主題的製作者很貼心，如果你需要客製化的話，在每個區塊都可以找到 custom.html 和 custom.scss 。

Favicon

我使用的是這個網站，製作好簡單的 icon 之後，將壓縮檔檔案整包放進 static 資料夾下，接著將下方語法貼入 assets/layout/partials/head 的 custom.html 裡，就可以得到小標了！

Comment system - Disqus

一個好的部落格應該要有留言板…吧，所以 Stack 也提供了完善的留言系統供我們選用，而我使用了第一個 Disqus。

首先可以到 Disqus 申請一個帳號，跟著跑一個 install on website 的流程，但我們的最終目的，只需要拿到General settings 頁面裡的 shortname ，接著更改 config.yaml 裡的 disqusShortname，就可以開啟留言版的服務了。

需要注意的是，我們可以在 Disqus 的頁面中設定留言版的一些功能，像是有沒有表情符號可以選、多久會關閉留言功能等。

部署網站 Deploy Website

關於部署網站的部分，我們會有兩個遠端數據庫，一個是在網站目錄下的.git，我自己是設定成 private repo，另一個是在我們利用指令 hugo，讓程式 generate 靜態網站到 public 資料夾裡，所以我們會在 public 資料夾內建一個 .git，設定成 public repo。

官方的部署資料內有許多關於其他平台上的部署方式，但我使用的是 GitHub Pages，也就是說，我接下來會把剛剛產生出靜態網站、放在 public 資料夾內的東西，全部 push 到 GitHub 上。

我們可以使用<USERNAME>.github.io作為這個公開的 repo 名稱，在 GitHub Pages 的規範裡，這是使用者或團體的頁面，與一般專案的 repo 不同。

我們現在已經有了設定好的網站，可以先到 GitHub 開啟一個新的、名為 <USERNAME>.github.io 的 repo，接著照著下面的步驟走

# 獲取 Repository URL，例如 https://github.com/username/username.github.io.git

# 進入到專案底下，打包你的網站，生成 public 資料夾
$ hugo

# 進入到 public 資料夾裡，執行遠端資源庫地址註冊
$ cd public
$ git init
$ git remote add https://github.com/username/username.github.io.git

# 確認遠端資源庫地址註冊成功
$ git remove -v

# 將目前異動提交成一個 commit
$ git add .
$ git commit -m 'initial site'

# 推送 commit
$ git push

如果之後新增文章需要更新網站，就可以在重新生成之後，把更動 commit 、 push 到這個公開的 repo 了。

於是乎，我們就可以在自己的 Pages 上看到個人部落格啦！
